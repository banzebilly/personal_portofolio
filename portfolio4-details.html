Pyhton: Inventory Management System for Shoes

This project is a Python-based application designed to manage the inventory of shoes for a retail business. The program reads data from a text file, processes it, and provides various functionalities to view, update, and analyze the inventory, ensuring efficient stock management and preparation for presentations to managers.

  • Class Definition: The core of the application is the Shoe class, which includes the following attributes:
    • country: The country where the shoes are manufactured.
    • code: A unique code identifying each shoe product.
    • product: The name of the shoe product.
    • cost: The cost of the shoes.
    • quantity: The quantity of the shoes in stock.
    The class also defines the following methods:
    • get_cost: Returns the cost of the shoes.
    • get_quantity: Returns the quantity of the shoes.
    • __str__: Returns a string representation of the shoe object.
  • Data Storage: A list is used to store instances of shoe objects, providing a structured way to manage and manipulate the inventory data.
  • Functions: The following functions are defined outside the class to handle various inventory operations:
    • read_shoes_data: Opens the inventory.txt file, reads the data, creates shoe objects, and appends them to the shoes list. Implements error handling using try-except blocks.
    • capture_shoes: Allows the user to input data for a new shoe, creates a shoe object, and appends it to the shoes list.
    • view_all: Iterates over the shoes list and prints the details of each shoe, optionally organizing the data in a table format using Pythonâ€™s tabulate module.
    • re_stock: Identifies the shoe with the lowest quantity, prompts the user to update the stock quantity, and updates the inventory file accordingly.
    • search_shoe: Searches for a shoe by its code and returns the corresponding shoe object for printing.
    • value_per_item: Calculates the total value for each shoe item (value = cost * quantity) and prints this information for all shoes.
    • highest_qty: Determines the shoe with the highest quantity in stock and highlights it as being for sale.
  • User Menu: The main program includes an interactive menu that allows the user to execute each function within a while loop. The menu options include:
    • 1. Read shoes data
    • 2. Capture shoes
    • 3. View all shoes
    • 4. Re-stock shoes
    • 5. Search shoes
    • 6. Calculate value per item
    • 7. Display highest quantity shoe
    • 0. Exit
    The user can select an option to perform the corresponding inventory management operation.

This project effectively demonstrates the use of Python for managing an inventory system, providing a user-friendly interface for handling shoe stock data, updating quantities, calculating values, and generating reports. The implementation showcases practical applications of object-oriented programming, file handling, and user interaction design.
